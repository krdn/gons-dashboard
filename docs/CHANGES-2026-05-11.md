## 발견·수정한 production 버그

### Gmail Response body 이중 consume (PR #24)
- **증상**: Google CDN 의 502 HTML 응답 시 cron 이 opaque "Body is unusable" 에러로 죽음.
  operational log 에 해당 패턴이 보이면 본 버그가 원인일 가능성.
- **근본 원인**: `src/shared/api/gmail/errors.ts:classifyGmailError` 가 `response.json()` 호출 후
  catch 에서 `response.text()` 호출. Response body 는 한 번만 consume 가능해 두 번째가 throw.
- **수정**: text 로 한 번 읽고 `JSON.parse` 시도 → 실패 시 string 그대로. 기존 typeof 체크가
  string 을 거부해 자연 fallthrough → 깨끗한 `GmailClientError` 분류.
- **회귀 가드**: `tests/gmail-errors.test.ts` 의 "body 가 JSON 이 아니어도 throw 안 함" 케이스.
