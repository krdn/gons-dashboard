## 1. 배경과 목표

### 1.1 v0.2 종료 시점 한계

v0.2 lifetime narrative richer 작업이 운영 배포된 직후, 김석곤 1967-03-29 05:30 양력 남자 프로파일의 2026 세운 평가가 신한라이프 (대단히 좋다) 와 우리 시스템 (3학파 unfavorable 합의) 정반대로 나옴. 진단 결과:

- 양 시스템 모두 신약 판정 일치
- 차이의 핵심: 우리 `adapters/{ko,cn-ziping}/yongshin.ts` 는 신약이면 무조건 인성 용신, **종격(從格) 판정 분기 자체가 없음**
- 김석곤 명조 (인성=0, 식상 압도) 는 정통적으로 종아격 후보 → 2026 丙午(火=재성) 가 食傷生財 길운
- 부수: lifetime `saju_charts.strength` 의 단순 개수 룰과 yearly 의 자평 룰이 모순

### 1.2 목표

v0.3 spec 범위:
- **A1**: 종격(從格) 판정 룰 도입 (종아·종재·종살)
- **A2**: 격국-용신 분기 통합 (傷官·正官·財·偏印 4종)
- **B2**: lifetime/yearly strength 단일 알고리즘 통합

범위 외 (v0.4 이후): B1 가중치 점수, C1 5분 grade, C2 fixture 광범위 회귀, mangpai/jp 종격 적용.

### 1.3 의사결정 요약

- **접근법 B 안**: 공유 모듈 추출 (`lib/shen-strength.ts`, `lib/jonggyeok.ts`, `lib/gyeokguk-yongshin.ts`) — ko/cn-ziping 두 학파가 공통 사용. A 안 (builder 내부 분기) 은 중복 우려, C 안 (데이터 주도) 은 과조.
- **종격 엄격도**: 완전종 + 가종 모두. 김석곤 같은 경계 case 통과.
- **격국 분기**: 핵심 4 격국 × 2 신강도 = 8 분기. 종격 우선.
- **학파 적용**: ko + cn-ziping 만. mangpai/jp 는 v0.2 그대로.
- **strength 통합 방향**: yearly 의 자평 룰로 lifetime 교체. `elements.ts:computeStrength` 의 단순 개수 룰 제거.
- **캐시 무효화**: `ALGORITHM_VERSION` 키 도입 (v0.2 의 `PROMPT_VERSION` 패턴 그대로).
- **종격 스키마**: `basisShenStrength` enum 확장 (신강·신약·균형·종아·종재·종살 6값).
- **지장간 카운트**: 종격 트리거 한정으로 도입. 다른 곳은 주기만.

## 2. 신규 공유 모듈 3종

### 2.1 `packages/saju/src/lib/shen-strength.ts` (신규)

자평 룰의 단일 신강도 알고리즘 + 종격 판정 통합. 기존 yongshin builder 와 elements.ts 두 곳에서 같은 함수 import.

```typescript
import type { SajuChart } from "../types";
import type { Element } from "../hanja";

export type Role = "비겁" | "인성" | "식상" | "재성" | "관성";

export type ShenVerdict =
  | "신강" | "신약" | "균형"
  | "종아" | "종재" | "종살";

export interface ShenStrengthResult {
  dayElement: Element;
  supportScore: number;           // 비겁 + 인성 (주기만)
  drainScore: number;             // 식상 + 재성 + 관성 (주기만)
  roleCount: Record<Role, number>;  // 주기만 (8글자 또는 6)
  roleCountExtended: Record<Role, number>;  // 주기 + 지장간
  verdict: ShenVerdict;
  jonggyeokKind: "완전종" | "가종" | null;
  jonggyeokRole: Role | null;     // 따르는 역할 (식상/재성/관성)
}

export function classifyRole(dayEl: Element, target: Element): Role;

export function computeShenStrength(chart: SajuChart): ShenStrengthResult {
  // 1. 8글자 (또는 6) 주기 카운트 → roleCount, support, drain
  // 2. 지장간까지 포함한 확장 카운트 → roleCountExtended
  // 3. detectJonggyeok(roleCountExtended) → 종격 판정
  // 4. 종격이면 verdict = 종아/종재/종살, jonggyeokKind/jonggyeokRole 설정
  // 5. 종격 아니면 자평 룰: support-drain >= 2 → 신강, drain-support >= 2 → 신약, else 균형
}
```

### 2.2 종격 트리거 룰 (shen-strength.ts 내부)

```typescript
function detectJonggyeok(
  roleCount: Record<Role, number>,    // 지장간 포함 확장 카운트
): { kind: "완전종" | "가종"; role: Role } | null {
  const { 비겁, 인성, 식상, 재성, 관성 } = roleCount;
  const dayHelperScore = 비겁 + 인성;

  // 종격은 식상·재성·관성 중 하나로만 따라감
  const dominant = [
    { role: "식상" as const, count: 식상 },
    { role: "재성" as const, count: 재성 },
    { role: "관성" as const, count: 관성 },
  ].reduce((max, c) => c.count > max.count ? c : max);

  // 완전종 — 매우 엄격
  if (인성 === 0 && 비겁 <= 1 && dominant.count >= 6) {
    return { kind: "완전종", role: dominant.role };
  }

  // 가종 — 중간 엄격도
  // 트리거: 인성 ≤1, dominant ≥ dayHelper × 1.3, dominant ≥ 4
  // 1.3 은 magic number — 김석곤 case (dayHelper=3, dominant=4) 통과 목적
  // 향후 fixture 늘리면서 보정 가능한 임시값
  if (
    인성 <= 1 &&
    dominant.count >= dayHelperScore * 1.3 &&
    dominant.count >= 4
  ) {
    return { kind: "가종", role: dominant.role };
  }

  return null;
}
```

```typescript
import type { Element } from "../hanja";
import type { ShenStrengthResult, Role } from "./shen-strength";

export interface JonggyeokYongshin {
  primary: Element;            // 따르는 오행 자체
  gisin: Element[];            // 거스르는 오행 (일간 + 인성)
  rationale: string;
}

export function buildJonggyeokYongshin(
  shen: ShenStrengthResult,
): JonggyeokYongshin | null {
  if (!shen.jonggyeokKind || !shen.jonggyeokRole) return null;

  // primary = 따르는 오행 (역할에 해당하는 element)
  // 종아: 일간이 생하는 오행 (식상)
  // 종재: 일간이 극하는 오행 (재성)
  // 종살: 일간을 극하는 오행 (관성)
  // gisin = 일간 오행 + 인성 오행

  // rationale 예: "인성 0·비겁 3 약함, 식상 4 우세 — 가종 종아격으로 식상의 흐름을 따른다"
}
```

## 3. 김석곤 검증 (트리거 동작 확인)

명조 4기둥: 丁未·癸卯·壬辰·癸卯, 일간 壬水.

**주기 카운트** (일간 포함, 8글자):
- 비겁=3 (壬·癸·癸), 인성=0, 식상=2 (卯·卯), 재성=1 (丁), 관성=2 (未·辰)
- support=3, drain=5 → support-drain=-2 → **신약**

**지장간 포함 확장 카운트**:
| 글자/장간 | 오행 | 역할 |
|---|---|---|
| 丁 | 火 | 재성 |
| 未(主) 土 | 관성 |
| 未(藏) 己 土 | 관성 |
| 未(藏) 丁 火 | 재성 |
| 未(藏) 乙 木 | **식상** |
| 癸 | 水 | 비겁 |
| 卯(主) 乙 木 | **식상** |
| 壬 (일간) | (제외) |
| 辰(主) 土 | 관성 |
| 辰(藏) 戊 土 | 관성 |
| 辰(藏) 乙 木 | **식상** |
| 辰(藏) 癸 水 | 비겁 |
| 癸 | 水 | 비겁 |
| 卯(主) 乙 木 | **식상** |

확장 roleCount: 비겁=3, 인성=0, 식상=4, 재성=2, 관성=4

**종격 트리거**:
- 인성=0 ≤1 ✓
- dayHelper=3, dominant 식상=4, 4 ≥ 3×1.3=3.9 ✓
- dominant=4 ≥ 4 ✓
- → **가종 종아격** ✓

**용신**: primary=木 (식상), gisin=[水(일간), 金(인성)]

**2026 丙午 평가**: 火 = 재성. 종아격에서 식상→재성 흐름은 길 → **favorable**.

기존 v0.2 (신약 → 인성 용신 金 → 火=기신 강화 → unfavorable) 와 정반대 결과. 신한라이프 평가와 일치.
